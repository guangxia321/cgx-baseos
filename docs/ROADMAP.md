# 广厦 BaseOS 迭代路线

## 第一阶段：结构精简与诊断框架

已完成：

- 明确“少就是多”的技能结构：主入口 + 诊断 + 构建 + 维修。
- 删除外显的经营财务、模板输出、飞书实施技能入口。
- 将经营财务、模板、看板新建、飞书 CLI 准备降级为构建内部资源。
- 建立一页业务数据地图，作为诊断到构建的流转产物。
- 将构建技能内部流程阶段化为基础数据层、录入与联动层、展示看板层、验收发布层。
- 更新打包脚本，只发布四个公开技能。

## 第二阶段：业务数据诊断稳定

接下来重点：

- 用一人公司、企业服务、销售型小企业案例校准诊断问题。
- 稳定“关键数据、重复动作、AI 可用数据、漏分析数据、录入方式”的判断。
- 明确什么时候不该搭系统，只输出轻量表格或 SOP。
- 形成 5-10 个可公开的一页业务数据地图样例。

## 第三阶段：构建和重构模板稳定

继续沉淀：

- 销售 / 客户 / 商品 / 订单 / 收款 / 经营看板。
- 一人公司产品、线索、客户、交付、收款、复购。
- 企业服务线索、商机、方案、合同、项目交付、续费、回款。
- 经营财务、应收、费用、利润、现金流。
- 旧表重构、历史 Excel 导入和批量录入校验流程。
- 看板新建：指标、数据源、筛选字段、组件类型和完整重载验证。

## 第四阶段：维修案例和回归测试

- 增加典型失败案例：字段混乱、看板不联动、公式错用、人工重复录入。
- 建立回归测试清单，防止技能迭代后重新走回“模块越拆越多”的旧路径。
- 补充飞书 Base 实际落地后的维修和验收案例。

## 第五阶段：开源发布准备

- 确认开源许可证。
- 补充贡献说明。
- 整理安装、调用和二次开发方式。
- 准备首批可公开的模板、诊断样例和演示案例。
