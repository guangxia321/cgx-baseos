# 广厦 BaseOS 开发背景

广厦 BaseOS（CGX BaseOS）会逐步完善为一个低门槛、可开源复用的经营数字化搭建技能，重点面向一人公司与企业服务两个大场景。

## 项目出发点

很多企业第一次接触 AI 时会发现 AI 很难真正用起来，根因往往不是模型不够强，而是企业内部还没有稳定的数据、标准流程和可复用的业务入口。

销售、订单、客户、商品 / 服务、项目交付、收款、财务、运营等信息如果仍然依赖人工口头同步、临时表格和个人经验，AI 就缺少可以理解、调用和维护的业务基座。

一人公司常见的问题是：个人品牌、内容资产、渠道账号、产品 / 服务、线索、客户、交付、收款、复购和财务没有被系统化管理，业务高度依赖创始人的脑子和临时记录。企业服务常见的问题是：从线索、商机、方案、合同，到项目交付、续费、收款和客户成功，都需要先变成稳定的数据结构。

## 为什么从多维表格开始

广厦 BaseOS 选择飞书多维表格作为默认起点，是因为它对小企业门槛低、上线快、协作成本低，并且适合承载第一套标准化经营流程。

但这个项目不是为了推广飞书，也不把能力限定在某个工具上。只要用户使用的是类似的低代码表格、SaaS 或轻量 ERP 载体，都应优先复用这里的建模原则。

## 开源目标

开源后的核心目标是让用户用尽量低的成本搭出第一套可用系统：先把关键业务流程数字化，再逐步沉淀 SOP、主数据、事实数据、公式、看板和自动化，最后让企业具备真正使用 AI 的基础条件。

这个项目会在真实业务场景中持续迭代，逐步沉淀可复用的行业模板、建模方法、排障经验和自动化流程。
