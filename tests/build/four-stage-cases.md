# 四阶段构建测试用例

这些用例用于检查 `cgx-baseos-build` 是否保持一个公开技能、四个内部阶段。

## 用例 1：销售系统四阶段输出

用户输入：

> 这是业务数据地图：客户、商品、订单、收款是主链路；我要搭销售系统并看销售额、利润、未收款和客户排行。

期望：

- 路由到 `cgx-baseos-build`。
- 输出必须分为基础数据层、录入与联动层、展示看板层、验收发布层。
- 基础数据层包含客户、商品/服务、订单、收款、财务流水。
- 录入与联动层包含日常录入页、批量录入页、Agent 入口、校验视图、自动字段、显式状态字段。
- 展示看板层包含数据源、指标字段、筛选字段、组件类型和验证方式。
- 验收发布层包含样例数据、完整重载和两个筛选值切换验证。

## 用例 2：不能跳过基础数据直接做看板

用户输入：

> 先帮我做一个经营看板，我还没确定订单、收款和客户表怎么设计。

期望：

- 仍路由到 `cgx-baseos-build`，不能路由到独立看板技能。
- 不能直接输出看板组件。
- 先要求确认基础数据层和录入与联动层。
- 明确看板是第三阶段。

## 用例 3：看板维修仍进 repair

用户输入：

> 我已经搭好的飞书看板月份筛选不动，指标卡没有变化。

期望：

- 路由到 `cgx-baseos-repair`。
- 读取 `dashboard-filter-slicer-case.md`。
- 不进入 build 第三阶段重新设计看板，除非维修确认根因是底层模型错误。

## 用例 4：模板必须阶段化

用户输入：

> 给我一个企业服务经营系统模板。

期望：

- 路由到 `cgx-baseos-build`。
- 读取 `templates/service-business/schema.md`。
- 模板必须包含基础数据层、录入与联动层、展示看板层、验收发布层。
- 必须区分用户录入字段、自动字段、显式状态字段和看板读取字段。
