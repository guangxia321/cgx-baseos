# 一页业务数据地图测试用例

这些用例用于检查 `cgx-baseos-diagnose` 的固定产物是否完整。每个输出都必须包含：关键业务对象、重复动作、已有数据、AI 价值、漏分析指标、录入方式、第一版构建范围。

## 用例 1：一人公司

用户输入：

> 我一个人做咨询服务，客户来自微信和小红书，订单、交付、收款都靠聊天记录和几个表格记。

期望地图：

- 业务类型：一人公司 / 咨询服务。
- 关键对象：产品/服务、渠道、线索、客户、订单/合同、交付记录、收款记录。
- 重复动作：记录线索、跟进客户、确认成交、安排交付、核对收款。
- AI 价值：按客户查询交付和收款状态、催收提醒、月度经营复盘、渠道转化分析。
- 录入方式：人手日常录入 + Agent 代录聊天信息 + 批量整理旧表。
- 第一版范围：产品/服务 -> 线索 -> 客户 -> 订单/交付 -> 收款 -> 经营看板。

## 用例 2：企业服务团队

用户输入：

> 我们是代运营团队，线索、报价、合同、项目进度、续费和回款分散在销售、交付、财务各自的表里。

期望地图：

- 业务类型：企业服务团队。
- 关键对象：客户、线索、商机、报价/方案、合同、项目、任务、收款、续费。
- 重复动作：线索跟进、报价、合同确认、项目交付、回款核对、续费提醒。
- AI 价值：项目风险提醒、逾期回款列表、客户成功摘要、续费机会识别。
- 录入方式：销售和交付日常录入 + 财务批量核对 + Agent 查询和提醒。
- 第一版范围：线索 -> 商机 -> 合同 -> 项目交付 -> 收款 -> 续费。

## 用例 3：销售型小企业

用户输入：

> 我们有固定客户、商品、订单和收款，每个月都要手工统计销售额、利润和未收款。

期望地图：

- 业务类型：销售型小企业。
- 关键对象：客户、商品/服务、订单、收款、费用、财务科目。
- 重复动作：登记订单、匹配商品、核对收款、统计利润、整理月报。
- AI 价值：客户销售排行、商品利润分析、未收款提醒、月度经营报告。
- 录入方式：日常订单录入 + Excel 批量导入 + 导入校验视图。
- 第一版范围：客户 / 商品 -> 订单 -> 收款 -> 费用 -> 月度经营看板。
