# Tests

这个目录用于存放技能评测和回归测试材料。

测试重点不是代码覆盖率，而是检查技能在真实业务问题中是否保持正确判断：

- 是否先诊断企业关键数据，而不是直接建表。
- 是否能输出一页业务数据地图。
- 是否优先降低用户录入成本。
- 是否区分主数据、事实数据、派生数据和展示数据。
- 是否把模板、经营财务、看板新建、飞书 CLI 当作构建内部能力，而不是独立技能。
- 构建输出是否按基础数据层、录入与联动层、展示看板层、验收发布层组织。
- 是否能先定位现有系统问题，而不是直接重建。
- 是否保留用户已有系统中已经跑通的结构。

## 第一版测试集

- `routing/first-version-cases.md`：检查主入口能否正确区分诊断、构建和维修，并验证一页业务数据地图的最低要求。
- `diagnosis/business-data-map-cases.md`：检查一人公司、企业服务、销售型小企业三类输入能否产出合格的一页业务数据地图。
- `build/four-stage-cases.md`：检查构建技能是否按四阶段输出，且看板新建留在 build 第三阶段。
