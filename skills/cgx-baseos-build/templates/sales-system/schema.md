# 销售 / 订单 / 收款系统模板

## 适合谁

适合有稳定客户、商品/服务、订单和收款动作的小企业、一人公司和销售型团队。目标是用低录入成本跑通：

客户 / 商品或服务 -> 订单 -> 收款 -> 利润 -> 经营看板

不适合一上来承载完整 ERP、复杂库存、复杂审批或专业会计。

## 阶段 1：基础数据层

| 层级 | 表 | 作用 |
| --- | --- | --- |
| 主数据 | 客户管理 | 客户名称、联系人、来源、状态、地址 |
| 主数据 | 商品 / 服务 | 名称、规格、售价、成本、利润率、状态 |
| 主数据 | 财务科目 | 收入、成本、费用、退款等分类 |
| 事实 | 订单录入 | 唯一订单事实入口；一单多商品可多行同订单号 |
| 事实 | 收款记录 | 多次收款、部分收款、凭证附件、收款渠道 |
| 事实 | 财务流水 | 支出、费用、非订单收入、退款 |
| 派生 | 订单汇总 | 按订单号聚合订单金额、收款、利润和状态 |
| 派生 | 月度经营汇总 | 按月份聚合收入、支出、利润、订单数 |

## 阶段 2：录入与联动层

日常录入页：

- 新客户：客户名称、联系人、来源、状态。
- 新商品 / 服务：名称、类型、标准售价、标准成本、状态、说明。
- 新订单：订单号、订单日期、客户、商品/服务、数量、成交价或折扣、备注/附件。
- 新收款：收款日期、关联订单、收款金额、收款渠道、凭证附件。
- 新支出：发生日期、类型、财务科目、金额、凭证附件。

主数据视图：

- 客户登记 / 维护视图：用于新增和维护客户，不和订单事实混在一起。
- 商品 / 服务登记视图：用于新增产品或服务，展示名称、类型、售价、成本、状态、说明。
- 可售商品 / 服务目录视图：只显示状态为上架的商品或服务，给订单录入和 Agent 查询使用。
- 商品 / 服务批量导入视图：字段顺序固定为名称、类型、标准售价、标准成本、状态、说明。
- 如果由外部协作者或手机端提交产品资料，在同一张商品 / 服务表上创建表单入口，不新建重复的产品录入表。

批量录入页：

- 字段顺序固定为订单号、订单日期、客户、商品/服务、数量、成交价、备注。
- 自动字段隐藏或放在后面，不要求用户粘贴公式、lookup 和汇总字段。

自动字段：

- 自动月份
- 自动商品/服务信息
- 自动应收金额
- 自动成本
- 自动利润
- 自动实收金额
- 自动未结清金额
- 订单计数字段

显式状态字段：

- 结算状态：未收款 / 部分收款 / 已结清 / 已取消
- 是否逾期
- 月份筛选

工作流：

- 新订单后补订单汇总行。
- 新订单或财务流水后补月度经营汇总行。
- 新收款后更新订单结算状态。
- 未收款或逾期提醒。

## 阶段 3：展示看板层

| 组件 | 数据源 | 指标字段 | 筛选字段 |
| --- | --- | --- | --- |
| 本月销售额 | 订单录入 / 订单汇总 | 自动应收金额 | 月份筛选 |
| 本月订单数 | 订单录入 | 订单计数字段 | 月份筛选 |
| 本月利润 | 订单录入 / 月度经营汇总 | 自动利润 | 月份筛选 |
| 未收款金额 | 订单汇总 | 自动未结清金额 | 结算状态、月份筛选 |
| 客户销售排行 | 订单录入 | 自动应收金额 | 月份筛选、客户 |
| 商品 / 服务排行 | 订单录入 | 自动应收金额 | 月份筛选、商品/服务 |
| 费用分类 | 财务流水 | 金额 | 月份筛选、财务科目 |

看板优先读取事实表和稳定汇总表，不默认新增看板缓存表。

## 阶段 4：验收发布层

- 新增客户、商品、订单和收款后，看板能联动。
- 一单多商品时订单数不被放大。
- 部分收款时能显示未结清和结算状态。
- 用户日常视图只展示需要填写的字段。
- 自动字段不用用户维护。
- 看板完整重载后仍渲染，并用两个不同月份验证指标变化。
