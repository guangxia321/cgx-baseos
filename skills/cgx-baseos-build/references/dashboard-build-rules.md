# 看板构建规则

## 何时读取

当用户要新建经营看板、销售看板、收款看板、项目看板、财务看板，或构建技能进入“展示看板层”时读取这份 reference。

本文件只处理新建看板设计和上线验收。已有看板报错、筛选不动、配置数据发生变更时，切到 `$cgx-baseos-repair`，优先读取 `dashboard-troubleshooting.md` 和 `dashboard-filter-slicer-case.md`。

## 进入条件

不要在基础数据和录入联动未稳定前设计看板。进入看板阶段前必须确认：

- 主数据和事实数据已经确定。
- 用户录入字段、自动字段、lookup 字段和显式状态字段已经确定。
- 至少有 3-5 条样例数据，或有可人工核对的历史数据。
- 关键筛选维度已经结构化，例如月份筛选、结算状态、项目状态、负责人、客户分层。

## 设计顺序

1. 先确定经营问题：用户看这个看板要做什么判断。
2. 再列指标：销售额、订单数、利润、未收款、交付状态、费用结构等。
3. 再定数据源：每个组件读哪张事实表或汇总表。
4. 再定字段：指标字段、筛选字段、分组字段、排序字段。
5. 最后选组件：指标卡、趋势、排行、分布、异常列表。

不要先画组件，再倒推字段。

## 数据源规则

看板优先读取：

1. 同一张事实表中的自动字段。
2. 稳定汇总表中的自动聚合字段。
3. 主数据表中的稳定分类字段。

同一个筛选器要控制多个核心组件时，优先让这些组件读取同一张事实表。不要假设同名字段可以跨表联动。

默认不要新增看板缓存表。只有在原生看板确实无法表达、并且用户接受维护成本时，才考虑额外中间表。

## 筛选字段规则

筛选字段优先使用：

- 单选字段。
- 多选字段。
- checkbox。
- 显式状态字段。
- 月份筛选字段。
- 稳定负责人或客户分层字段。

不优先使用：

- 手填文本。
- 复杂公式文本。
- lookup 结果。
- 关联展开字段。
- 复杂公式数字比较。

金额字段回答“多少钱”，状态字段回答“算什么状态”。不要只用 `待收金额 > 0` 作为未收款筛选。

## 常见看板

销售 / 订单看板：

- 指标卡：销售额、订单数、利润、未收款。
- 趋势：月度销售额、回款、利润。
- 排行：客户排行、商品/服务排行、负责人排行。
- 异常：未结清订单、逾期应收。

企业服务看板：

- 销售漏斗金额。
- 本月签约额和回款。
- 项目状态分布。
- 风险项目列表。
- 续费提醒。
- 项目利润。

一人公司看板：

- 本月成交额。
- 本月回款。
- 待交付 / 逾期交付。
- 产品/服务销售排行。
- 渠道来源转化。
- 月度利润。

经营财务看板：

- 本月收入。
- 本月支出。
- 本月经营利润。
- 未收款金额。
- 费用分类排行。
- 月度收入 / 支出 / 利润趋势。

## 上线验收

看板上线必须验证：

- 服务端组件配置能读取。
- 页面完整重载后组件仍正常渲染。
- 每个核心组件的数据源、指标字段、筛选字段和分组字段明确。
- 至少两个筛选值切换后，目标组件数值发生正确变化。
- 新增样例数据后，指标和排行会联动。
- 没有要求用户手工维护汇总表或缓存表。
- 临时测试组件、字段、视图已清理。

如果看板上线时发现筛选不动、组件空白或配置变更，停止构建阶段，切到 `$cgx-baseos-repair` 做最小排障。
