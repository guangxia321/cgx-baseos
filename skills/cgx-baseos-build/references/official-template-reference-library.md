# 飞书官方模板参考库

## 何时读取

当用户需要参考飞书官方多维表格模板、寻找行业模板、拆解模板结构，或希望把官方模板沉淀进广厦 BaseOS 时，读取这份 reference。

这份资料不是业务模板本身，而是官方模板复制件的索引入口。实际模板已经复制到当前飞书账号的参考库目录，并按互联网可读、可复制的目标设置了公开权限。

如果目标是搭建或重构系统，先读 `official-template-design-patterns.md`。本文件回答“有哪些具体官方模板可以参考”，设计模式文件回答“这些模板共同证明了哪些结构应该被 BaseOS 吸收”。

## 分类方法

不要照搬飞书官网的行业分类。参考库按 BaseOS 建模价值分为：

- 通用经营底座：CRM、线索、商机、客户、合同、订单、回款、经营看板。
- 销售订单合同：销售订单、合同、发票、收款、收入统计。
- 项目交付经营：项目、任务、交付、里程碑、回款、成本利润。
- 财务经营：预算、费用、报销、发票、经营明细。
- 进销存供应链：商品、采购、供应商、仓库、库存流水。
- 门店餐饮零售：门店、餐饮、会员、预约、开单、巡检、连锁库存。
- 电商直播内容：电商运营、直播复盘、内容生产、达人/KOL、选品。
- 制造生产：生产任务、工序、排产、工厂看板、质量巡检。
- 售后服务：工单、投诉、退换货、满意度、客服质检。
- 教育专业服务：课程销售、教培客户、案件、翻译项目。

## 使用规则

- 官方模板索引用于找具体案例；`official-template-design-patterns.md` 用于抽象系统结构和搭建规则。
- 先读索引 Base，不要从官网重新盲选模板。
- 参考库不是封闭集合。如果索引 Base 里没有合适模板，或用户的问题明显属于更细行业场景，回到飞书官方模板库用关键词搜索，再判断是否需要复制、公开并补充进索引。
- 官方模板库有搜索栏和大量细分模板，适合按行业词、业务词或对象词检索，例如餐饮、巡检、会员、采购、工单、预算、直播、教培、律所、库存。
- 深拆模板时，优先看表名、字段、视图、看板和工作流，不要只看官方描述。
- 吸收结构时，按 BaseOS 的四层数据模型重构：主数据、事实数据、派生数据、展示数据。
- 行业模板不要整套照搬。先抽象通用对象，例如客户、商品/服务、订单、项目、合同、收款、支出、库存、任务。
- AI 或自动化模板要先拆数据结构，再判断自动化动作是否真的需要进入第一版系统。
