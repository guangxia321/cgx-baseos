# 经营财务场景

## 何时读取

当用户提到收款、应收、欠款、支出、费用、利润、现金流、凭证、发票、月报、项目利润时，读取这份 reference。

这里的财务指“经营财务”，目标是让小企业看清业务收入、支出、利润和应收，不是替代专业会计、报税、总账或复式记账。

## 财务场景分类

| 场景 | 用户真实问题 | 第一版系统 |
| --- | --- | --- |
| 收款管理 | 谁还没付款、欠多少、什么时候该催 | 订单、收款记录、结算状态、未收款看板 |
| 支出管理 | 钱花到哪里了、哪些费用太高 | 财务流水、财务科目、费用看板 |
| 利润核算 | 这单/这个项目到底赚了多少 | 订单利润、项目成本、项目利润 |
| 现金流 | 本月进了多少钱、出了多少钱、还剩多少 | 收入流水、支出流水、月度经营汇总 |
| 凭证管理 | 截图、发票、合同、收据到处都是 | 附件字段、凭证类型、关联订单/客户 |
| 月度经营报表 | 每个月销售额、回款、费用、利润 | 月度汇总、经营看板、月报视图 |
| 项目财务 | 一个客户或项目投入产出怎么算 | 项目、收款节点、成本流水、项目利润 |

## 推荐表结构

| 层级 | 表 | 说明 |
| --- | --- | --- |
| 主数据 | 财务科目 | 收入、成本、费用、税费、退款等分类 |
| 主数据 | 账户 / 收款渠道 | 银行卡、支付宝、微信、对公账户等 |
| 事实 | 收款记录 | 每一笔真实收款，保留金额、日期、渠道、凭证 |
| 事实 | 财务流水 | 支出、费用、非订单收入、退款、调整 |
| 事实 | 订单 / 项目 | 财务记录的业务来源 |
| 派生 | 月度经营汇总 | 按月份聚合收入、支出、利润、未收款 |
| 展示 | 财务看板 | 回款、费用、利润、现金流、逾期应收 |

## 字段设计要点

收款记录常见输入字段：

- 收款日期
- 客户
- 关联订单 / 项目
- 收款金额
- 收款渠道
- 凭证附件
- 备注

财务流水常见输入字段：

- 发生日期
- 类型：收入 / 支出 / 退款 / 调整
- 财务科目
- 金额
- 关联客户 / 订单 / 项目
- 支付渠道
- 凭证附件
- 备注

常见自动字段：

- 自动月份
- 自动收入金额
- 自动支出金额
- 自动经营利润
- 自动未结清金额
- 自动结算状态
- 项目累计收入
- 项目累计成本
- 项目利润

## 状态字段

财务系统最容易出问题的是把状态藏在金额比较里。应优先建立显式状态：

- 结算状态：未结清 / 部分结清 / 已结清 / 退款中
- 是否逾期：是 / 否
- 凭证状态：未上传 / 已上传 / 待补充
- 入账状态：待确认 / 已确认 / 需调整

金额字段回答“多少钱”，状态字段回答“算什么状态”。不要只靠 `未收金额 > 0` 表达业务状态。

## 看板指标

第一版财务看板优先包含：

- 本月收入
- 本月支出
- 本月经营利润
- 未收款金额
- 逾期应收
- 费用分类排行
- 客户回款排行
- 月度收入 / 支出 / 利润趋势

看板优先读取事实表自动字段和显式状态字段。不要默认新增看板缓存表。

## 需要先本地验算的情况

以下场景先用本地 Excel / 表格验算，再落到飞书 Base：

- 利润分成规则复杂。
- 一个订单对应多次收款、多次退款、多方成本。
- 项目成本需要按人工、外包、广告、平台费分摊。
- 历史财务数据很多，且字段口径不一致。
- 错误结果会影响对账、收款或对外报告。

## 边界

第一版不要默认做：

- 完整会计科目体系。
- 复式记账。
- 税务申报。
- 发票全生命周期。
- 多公司合并报表。
- 复杂权限审批。

如果用户明确需要这些能力，先确认是否已有专业会计或财务软件，再判断广厦 BaseOS 是做经营辅助系统，还是只做数据入口和看板。
